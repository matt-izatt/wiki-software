# Memory Management

### Overview

Memory management is the operating system function responsible for controlling and coordinating the use of main memory (RAM). It ensures that processes have the memory they need to execute while preventing them from interfering with each other.

Modern operating systems allow multiple processes to run simultaneously. Memory management enables this by allocating memory to processes, tracking memory usage, protecting address spaces, and moving data between RAM and secondary storage when necessary.

Without effective memory management, processes could overwrite each other's data, waste memory resources, or cause system instability.

***

### Goals of Memory Management

The primary objectives of memory management are:

#### **Efficient Memory Utilisation**

RAM is a limited resource. The operating system should use available memory efficiently while minimising waste.

#### **Process Isolation and Protection**

Each process should operate within its own address space and be prevented from accessing memory belonging to other processes.

#### **Relocation**

Programs should be able to run regardless of where they are loaded in physical memory.

#### **Sharing**

Memory should support controlled sharing between processes when required, such as shared libraries and inter-process communication.

#### **Virtualisation**

Processes should experience the illusion of having a large, contiguous memory space even when physical memory is fragmented or limited.

***

### Address Binding

When a program executes, it uses memory addresses to access instructions and data.

Three types of addresses are involved:

#### **Logical Address**

Also called a virtual address.

Generated by the CPU and used by the running program.

#### **Physical Address**

The actual location in RAM.

#### **Address Binding**

The process of mapping logical addresses to physical addresses.

Binding can occur at:

* Compile Time
* Load Time
* Execution Time

Modern operating systems typically perform binding during execution using hardware support such as the Memory Management Unit (MMU).

***

### Logical vs Physical Address Space

A process operates within a logical address space.

The operating system and MMU translate these logical addresses into physical memory locations.

Benefits include:

* Process isolation
* Memory protection
* Easier relocation
* Support for virtual memory

Applications are unaware of where their data physically resides in RAM.

***

### Memory Allocation

The operating system must decide how memory is allocated to processes.

#### **Contiguous Allocation**

Each process receives one continuous block of memory.

Advantages:

* Simple implementation
* Fast access

Disadvantages:

* External fragmentation
* Difficult memory growth

#### **Non-Contiguous Allocation**

A process can be spread across multiple memory locations.

Advantages:

* Better memory utilisation
* Supports virtual memory

Disadvantages:

* More complex implementation

Modern operating systems primarily use non-contiguous allocation.

***

### Fragmentation

Fragmentation occurs when memory becomes inefficiently organised.

#### **Internal Fragmentation**

Unused space exists inside allocated memory blocks.

Example:

A process requests 10 KB but receives a 12 KB block.

2 KB is wasted.

#### **External Fragmentation**

Free memory exists but is split into many small blocks.

Although enough total memory may exist, there may not be a sufficiently large contiguous block available.

#### **Compaction**

The operating system can reduce external fragmentation by moving processes and combining free space into larger blocks.

This operation is expensive and therefore used sparingly.

***

### Paging

Paging is the most common memory management technique used by modern operating systems.

Memory is divided into fixed-size blocks:

* Pages (logical memory)
* Frames (physical memory)

When a process accesses memory:

1. The CPU generates a virtual address.
2. The MMU checks the page table.
3. The virtual page number is translated into a physical frame number.
4. The physical address is produced.

Advantages:

* Eliminates external fragmentation.
* Simplifies memory allocation.
* Enables virtual memory.

Disadvantages:

* Page table overhead.
* Additional address translation cost.

***

### Page Tables

A page table stores mappings between virtual pages and physical frames.

Each process has its own page table.

A page table entry typically contains:

* **Frame number** — Identifies the physical memory frame where the page is stored.
* **Valid/invalid bit** — Indicates whether the page is currently mapped to a valid memory location.
* **Protection bits** — Define the access permissions for the page, such as read, write, or execute.
* **Dirty bit** — Shows whether the page has been modified since it was loaded into memory.
* **Reference bit** — Records whether the page has been accessed recently.

The operating system uses page tables to enforce protection and perform address translation.

***

### Translation Lookaside Buffer (TLB)

Address translation using page tables can be slow.

A Translation Lookaside Buffer (TLB) is a small, high-speed cache that stores recent page table entries.

When a memory access occurs:

1. Check the TLB.
2. If found (TLB hit), translation is immediate.
3. If not found (TLB miss), consult the page table.

The TLB significantly improves memory access performance.

***

### Virtual Memory

Virtual memory allows processes to use more memory than is physically available in RAM.

Only actively used pages remain in memory.

Inactive pages can be stored on disk.

Benefits:

* Larger address spaces.
* Better multiprogramming.
* Efficient memory utilisation.

The process sees a large continuous memory space regardless of actual RAM availability.

***

### Demand Paging

With demand paging, pages are loaded only when they are needed.

If a process accesses a page that is not currently in memory:

1. A page fault occurs.
2. The operating system pauses the process.
3. The required page is loaded from disk.
4. Execution resumes.

Demand paging reduces memory usage but page faults are expensive.

***

### Page Replacement Algorithms

When RAM is full and a new page must be loaded, the operating system must choose a page to remove.

Common algorithms include:

#### **FIFO (First-In, First-Out)**

Removes the oldest page.

Simple but may perform poorly.

#### **Optimal**

Removes the page that will not be used for the longest time in the future.

Provides the best theoretical performance but cannot be implemented perfectly.

#### **LRU (Least Recently Used)**

Removes the page that has not been used for the longest time.

Widely used because recent usage often predicts future usage.

***

### Thrashing

Thrashing occurs when a system spends more time handling page faults than executing processes.

Symptoms:

* High disk activity
* Low CPU utilisation
* Poor performance

Causes:

* Too many active processes
* Insufficient RAM

The operating system can reduce thrashing by lowering the degree of multiprogramming or allocating more memory.

***

### Memory Protection

Memory protection prevents processes from accessing unauthorised memory regions.

Common techniques include:

* Base and limit registers
* Page-level protection bits
* Read/write/execute permissions

Protection improves system reliability and security.

***

### Shared Memory

Operating systems can allow multiple processes to access the same physical memory region.

Benefits:

* Fast inter-process communication
* Reduced duplication of data
* Shared libraries loaded once and used by many processes

Access controls ensure processes only share memory intentionally.

***

### Summary

Memory management is responsible for allocating, protecting, and organising memory resources.

Key concepts include:

* Logical and physical addresses
* Paging and page tables
* Virtual memory
* Demand paging
* Page replacement algorithms
* Fragmentation
* Memory protection
* Thrashing

Together, these mechanisms allow modern operating systems to run many processes efficiently, safely, and reliably.
